# Личный бюджет

Веб-приложение для ведения месячного бюджета: доходы и расходы по категориям, аналитика по месяцу и за год, диаграммы. Запускается локально, данные хранятся у тебя на диске.

- **Бэкенд:** Go  
- **Фронт:** HTML, CSS, JavaScript (без npm)  
- **Данные:** `data.json` (все месяцы и операции), `categories.json` (категории)

## Как пользоваться (иконка в Dock / Программы)

1. Запускаешь приложение **иконкой** (см. ниже, как сделать иконку).
2. Открывается браузер на странице приложения.
3. Работаешь с бюджетом в браузере.
4. **Закрываешь вкладку** — сервер сохраняет данные и полностью завершается. Консоль не нужна.

При следующем запуске по иконке снова поднимется сервер и откроется браузер.

## Возможности

- **Главная** — список месяцев по годам (вкладки), автосоздание нового месяца по московскому времени, удаление месяца.
- **Страница месяца** — доходы/расходы по дням и категориям, диаграммы, остаток за месяц.
- **Аналитика за год** — сводка по году, остаток по месяцам, накоплено за год, диаграммы по категориям и месяцам.
- **Категории** — настройка категорий расходов и доходов прямо в приложении (без правки кода).
- **Бэкап** — кнопка «Скачать бэкап» в шапке: скачивается файл с данными. На новом компе или в новом клоне проекта положи файл в папку — данные подхватятся.
- **Поиск** — по описанию, категории, сумме по всем месяцам.

## Требования

- [Go](https://go.dev/) 1.16+

## Запуск из терминала

```bash
git clone https://github.com/YOUR_USERNAME/budget.git
cd budget
go build -o budget .
./budget
```

Открой в браузере **http://localhost:3001**. При **закрытии вкладки** сервер сохраняет данные и завершает работу.

## Иконка в Dock или в «Программы» (Mac)

Есть два способа получить иконку, по которой можно запускать приложение без терминала. В обоих случаях: жмёшь на иконку → открывается браузер с приложением; закрываешь вкладку → сервер сохраняет и выключается.

### Вариант 1: Файл в папке проекта (можно закрепить в Dock)

1. Собери бинарь (один раз):
   ```bash
   cd /путь/к/budget
   go build -o budget .
   ```

2. Дай права скрипту (один раз):
   ```bash
   chmod +x start-budget.command
   ```

3. Дважды кликни по **`start-budget.command`**. Откроется окно Терминала и браузер. Работаешь в браузере; когда закроешь вкладку с приложением — сервер сохранит данные и завершится.

**Чтобы окно Терминала закрывалось само** после закрытия вкладки (без вопроса «Прервать процессы?»), один раз включи в Терминале: **Терминал → Настройки → Профили → Оболочка** → поставь галку **«Закрывать окно при корректном завершении команды»** (или «Close the window if the shell exited cleanly»). После этого при закрытии вкладки сервер завершится, команда закончится — и окно Терминала закроется автоматически.

Файл **`start-budget.command`** можно перетащить в Dock — будет иконка для быстрого запуска.

### Вариант 2: Приложение в «Программы» (Launchpad, без окна Терминала)

Чтобы в списке программ была отдельная программа «Личный бюджет» (без консоли):

1. Собери бинарь: `go build -o budget .`
2. Открой **Automator** (Программы → Утилиты).
3. «Новый документ» → **Программа** (Application).
4. Слева: **Действия** → **Утилиты** → перетащи **«Запустить оболочный сценарий»** в правую часть.
5. В поле сценария вставь (подставь **свой путь** к папке проекта):

```bash
cd /ПУТЬ/К/ПАПКЕ/budget
./budget &
BID=$!
sleep 1
open "http://localhost:3001"
wait $BID
```

6. **Файл → Сохранить** — сохрани как «Личный бюджет» в папку **Программы** (Applications).

После этого приложение можно запускать из Launchpad или из папки «Программы». Окно Терминала не открывается. При закрытии вкладки с приложением сервер завершится и программа «остановится».

**Закрепить в Dock:** перетащи иконку «Личный бюджет» из папки Программы (или с рабочего стола) в левую часть Dock — туда, где стоят другие приложения. Иконка закрепится, запуск будет в один клик.

## Бэкап и перенос данных

В шапке приложения кнопка **«Скачать бэкап»** — скачивается файл с текущим содержимым `data.json`.

**Восстановление на другом компе или в новом клоне:**

1. Скопируй скачанный файл в папку проекта.
2. Переименуй его в **`data.json`** (или замени им существующий `data.json`).
3. Запусти приложение — все месяцы и операции отобразятся.

При желании можно так же сохранить и подставить **`categories.json`** (категории расходов/доходов).
